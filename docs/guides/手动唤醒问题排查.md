## 排查步骤

### 1. 确认 skills-admin 文件存在

```powershell
# 检查文件是否存在
Test-Path .cursor\skills\skills-admin\SKILL.md
```

应该返回 `True`。

### 2. 重新加载 Cursor

**重要**：修改 SKILL.md 后，Cursor 需要重新加载才能识别新的触发词。

**方法 1：重新加载窗口**
- 按 `Ctrl+Shift+P`
- 输入 `Developer: Reload Window`
- 回车

**方法 2：重启 Cursor**
- 完全关闭 Cursor
- 重新打开项目

### 3. 尝试不同的触发词

在 Cursor 聊天中依次尝试以下命令：

1. `审查建议`
2. `审查 proposals`
3. `review proposals`
4. `担任管理员`
5. `skills-admin`
6. `审查 pending proposals`
7. `审查待处理的提议`

### 4. 检查 Cursor 是否识别了 skill

在聊天中输入：
```
列出所有可用的 skills
```

或者：
```
show me available skills
```

看看是否包含 `skills-admin`。

### 5. 直接引用 skill

尝试更明确的命令：
```
使用 skills-admin skill 审查 .openskills/proposals/ 下的 pending proposals
```

或者：
```
Act as skills-admin and review pending proposals in .openskills/proposals/
```

### 6. 检查项目结构

确保你在正确的项目目录中：
- 项目根目录应该有 `.openskills` 文件夹
- 项目根目录应该有 `.cursor/skills/skills-admin/SKILL.md`

### 7. 查看 Cursor 日志

如果仍然不工作，查看 Cursor 的开发者工具：
- 按 `Ctrl+Shift+I` 打开开发者工具
- 查看 Console 标签页
- 查看是否有关于 skill 加载的错误信息
